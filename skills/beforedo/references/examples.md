# 示例

## 示例 1：有模糊点 → 澄清 → 最终确认
用户：我想新增“删除记录并清理资源”的功能。
助手（计划阶段）:
- 当下计划:
  - 明确删除入口与交互形式。
  - 设计删除范围与后端删除逻辑。
  - 加入确认弹窗与失败提示。
- LLM 的意见/看法:
  - 删除范围越大越需要二次确认，避免误删。
- 需要确认的模糊点:
  - 删除范围包含哪些资源？
  - 是否需要二次确认弹窗？

用户：删除范围包含会话记录和相关文件，需要弹窗确认。

助手（最终确认）:
- 最终计划:
  - 增加删除入口与确认弹窗。
  - 后端删除会话记录及关联文件。
  - 前端删除后刷新列表并提示结果。
- 确认执行:
  - 没问题请回复“确认执行”（或等价表达）以开始；如有问题请指出，我会回到计划阶段。

用户：确认执行
助手：开始执行。

## 示例 2：无模糊点 → 直接确认
用户：只需要按标题搜索，不要搜索内容。
助手（计划阶段，已了解项目不重复扫描）:
- 当下计划:
  - 添加一个仅按标题过滤的搜索输入。
  - 绑定过滤逻辑与结果展示。
- LLM 的意见/看法:
  - 标题搜索可以保持性能与交互简单。
- 需要确认的模糊点:
  - 无。

助手（最终确认）:
- 最终计划:
  - 添加标题搜索输入与过滤逻辑。
  - 更新列表渲染与空状态提示。
- 确认执行:
  - 没问题请回复“确认执行”（或等价表达）以开始；如有问题请指出，我会回到计划阶段。

用户：确认执行
助手：开始执行。
